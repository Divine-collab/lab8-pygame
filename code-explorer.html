🔄 Lifespan System NEW FEATURE

Dynamic population turnover creating realistic behavior.

How It Works

Frame 0: Create 10 squares
  Each square:
    - Gets random lifespan (60-180 frames)
    - Stored in self.lifespan
    - Remembered in self.birth_frame

Each Frame:
  For each square:
    - Calculate age = current_frame - birth_frame
    - Check: is age > lifespan?
    - If YES: Replace with new square
    - If NO: Keep alive

Result: Continuous population flow

Why Random Lifespan Matters

Aspect Fixed Lifespan Random Lifespan
Death Pattern All synchronized Staggered over time
Visual Effect Wave pattern (artificial) Natural flow
Realism Unrealistic Like real populations
Behavior Impact Jerky, repetitive Smooth, organic

🔁 Game Loop Architecture

Main execution cycle at 60 FPS with complete lifecycle management.

Execution Flow

Initialize Pygame
Create 10 initial squares via create_random_square(0)

MAIN LOOP (60 times per second):
  1. Increment frame counter
  2. Handle quit events
  3. Lifecycle Management:
     - Check each square: is_dead(frame)?
     - Replace dead with create_random_square(frame)
  4. Physics Updates:
     - For each square: update(all_squares)
     - For each square: check_boundaries()
  5. Rendering:
     - Clear canvas
     - Draw all squares
     - Update display
  6. Frame Timing:
     - Maintain 60 FPS via clock.tick(60)

Cleanup: pygame.quit()

Critical Order of Operations

  1. Lifecycle First: Check/replace dead squares
  2. Physics Second: Update living squares
  3. Render Last: Display updated state

Why? Ensures game state is consistent: dead squares never get updated or rendered.

⚙️ Core Algorithms

Mathematical and computational algorithms powering the simulation.

1. Threat Detection Algorithm

Goal: Find all squares larger than self within detection range

for each other_square in all_squares:
    if other_square is self:
        skip
    if other_square.size <= (self.size + SIZE_THRESHOLD):
        skip
    distance = sqrt((self.x - other.x)² + (self.y - other.y)²)
    if distance < DETECTION_RANGE:
        add to threats list
return threats

Complexity: O(n) where n = number of squares

2. Vector Calculation (Flee Force)

Goal: Calculate combined escape vector from all threats

for each threat:
    1. Calculate direction away: (self.x - threat.x, self.y - threat.y)
    2. Normalize: divide by distance to get unit vector
    3. Calculate urgency: (DETECTION_RANGE - distance) / DETECTION_RANGE
       Closer threat = higher urgency (0 to 1)
    4. Apply force: accumulate (normalized * urgency * FLEE_FORCE)
return accumulated (flee_x, flee_y)

Example: Square at (100,100) detecting threat at (150,150)

Direction: (-50, -50)
Normalized: (-0.707, -0.707)
Distance: 70.7px
Urgency: (100-70.7)/100 = 0.29
Force: -0.707 * 0.29 * 2.0 = -0.41
Result: Flee southwest

3. Speed Clamping

Goal: Ensure velocity never exceeds MAX_SPEED

speed = sqrt(velocity_x² + velocity_y²)
if speed > MAX_SPEED:
    scale = MAX_SPEED / speed
    velocity_x *= scale
    velocity_y *= scale

Key Property: Preserves direction while limiting magnitude

4. Jitter (Velocity Rotation)

Goal: Add organic randomness by rotating velocity vector

if random() < 0.05:  // 5% chance
    speed = sqrt(velocity_x² + velocity_y²)
    current_angle = atan2(velocity_y, velocity_x)
    rotation = random(-30°, 30°)
    new_angle = current_angle + rotation
    velocity_x = speed * cos(new_angle)
    velocity_y = speed * sin(new_angle)

Result: Direction changes, speed stays same

5. Lifecycle Management NEW

Goal: Track square age and trigger replacement

new_squares = []
for each square in squares:
    age = current_frame - square.birth_frame
    if age > square.lifespan:
        new_squares.append(create_random_square(current_frame))
    else:
        new_squares.append(square)
squares = new_squares

Result: Continuous population turnover
