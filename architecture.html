🏗️ Design Philosophy

HYBRID APPROACH Uses both frame-based and time-based systems:

  • Frame-Based: Lifecycle management, event handling, population turnover
  • Time-Based: Physics movement, velocity calculations, position updates
  • Result: Simple event logic + professional physics behavior

SIZE-BASED ROLES Agents automatically determined to be predator or prey:

  • Average size = (10 + 40) / 2 = 25 pixels
  • Size ≥ 25 px → Predator (hunts smaller squares)
  • Size < 25 px → Prey (flees from larger squares)
  • Behavior automatically switches based on role

🔧 Core Components

1. Square Class (Agent)

Represents an individual agent with independent behavior and state.

Attributes

Attribute Type Range Purpose
x, y float 0-800, 0-600 Position on canvas
size int 10-40 Physical dimensions
color tuple RGB (0-255) Visual appearance
velocity_x, velocity_y float -300 to 300 Speed px/second
birth_frame int 0+ Creation frame
lifespan int 60-180 Lifetime frames

Methods Overview

  • detect_larger_squares() - Find threats for prey
  • detect_smaller_squares() - Find prey for predators
  • calculate_flee_vector() - Escape force calculation
  • calculate_chase_vector() - Pursuit force calculation
  • update() - Main physics loop with role-based behavior
  • apply_jitter() - Organic movement randomization
  • check_boundaries() - Collision detection and bouncing
  • is_dead() - Lifespan checking
  • draw() - Rendering to surface

2. Game Loop

Central coordinator managing frame timing and system updates.

Execution Sequence

while running:
    frame += 1
    delta_time = clock.get_time() / 1000.0
    
    # Handle events
    for event in pygame.event.get():
        if event.type == pygame.QUIT: running = False
    
    # Lifecycle (frame-based)
    for square in squares:
        if square.is_dead(frame):
            replace with new_square
    
    # Physics (time-based)
    for square in squares:
        square.update(all_squares, delta_time)
        square.check_boundaries()
    
    # Rendering
    screen.fill(white)
    for square in squares:
        square.draw(screen)
    pygame.display.flip()
    
    clock.tick(60)

3. Helper Function

create_random_square(birth_frame)

  • Creates agents with random properties
  • Random size: 10-40 pixels
  • Speed inversely scaled to size (smaller = faster)
  • Random starting position and direction
  • Random color and lifespan (60-180 frames)

🎭 Behavior System

Role Determination

Average Size = (MIN_SIZE + MAX_SIZE) / 2 = 25 pixels

if agent.size >= 25:
    role = PREDATOR
    behavior = HUNT smaller squares
else:
    role = PREY
    behavior = FLEE from larger squares
Key Insight: Role is determined dynamically during each frame based on relative size. No agent is permanently predator or prey—it depends on who they're interacting with.

Predator Behavior

SIZE ≥ 25 PX

Algorithm

1. detect_smaller_squares()
   → Find all squares smaller than self
   → Within 100px detection range

2. calculate_chase_vector()
   → For each prey target:
      - Direction = toward target
      - Distance-based urgency: (100 - distance) / 100
      - Force = urgency * CHASE_FORCE (2.0)
   → Accumulate all chase forces

3. Apply to Velocity
   → velocity += accumulated_chase_vector

4. Clamp Speed
   → Limit to MAX_SPEED (300 px/s)

5. Update Position
   → position += velocity * delta_time

6. Add Jitter
   → 5% chance to rotate velocity ±30°

Characteristics

  • Actively hunts nearby prey
  • Multiple prey increase hunting force (accumulation)
  • Force increases with proximity
  • Max speed prevents excessive velocity
  • Jitter adds unpredictable pursuit patterns

Prey Behavior

SIZE < 25 PX

Algorithm

1. detect_larger_squares()
   → Find all squares larger than self
   → Within 100px detection range

2. calculate_flee_vector()
   → For each threat:
      - Direction = away from threat
      - Distance-based urgency: (100 - distance) / 100
      - Force = urgency * FLEE_FORCE (2.0)
   → Accumulate all flee forces

3. Apply to Velocity
   → velocity += accumulated_flee_vector

4. Clamp Speed
   → Limit to MAX_SPEED (300 px/s)

5. Update Position
   → position += velocity * delta_time

6. Add Jitter
   → 5% chance to rotate velocity ±30°

Characteristics

  • Actively evades nearby predators
  • Multiple threats increase evasion force
  • Force increases with proximity
  • Speed limited to prevent impossible escapes
  • Jitter enables unpredictable evasion

Detection Logic

detect_larger_squares()

For each other_square:
    if other_square is self: SKIP
    if other_square.size <= self.size + SIZE_THRESHOLD: SKIP
    distance = sqrt((dx)² + (dy)²)
    if distance < DETECTION_RANGE: ADD to threats
return threats

detect_smaller_squares()

For each other_square:
    if other_square is self: SKIP
    if other_square.size >= self.size: SKIP
    distance = sqrt((dx)² + (dy)²)
    if distance < DETECTION_RANGE: ADD to prey
return prey

⚙️ Physics System

Time-Based Movement

Movement is scaled by actual elapsed time, ensuring consistent speed across frame rates.

Formula

position += velocity * delta_time

At 60 FPS:
  delta_time = 1/60 = 0.0167 seconds
  If velocity = 300 px/s:
    movement = 300 * 0.0167 = 5 pixels per frame

At 30 FPS (if frame rate drops):
  delta_time = 1/30 = 0.0333 seconds
  movement = 300 * 0.0333 = 10 pixels per frame
  (Same speed, but half as often)
Benefit: Movement is FPS-independent. Speed remains constant whether running at 60 FPS or 30 FPS.

Speed Clamping

Prevents unlimited acceleration from force accumulation.

speed = sqrt(velocity_x² + velocity_y²)
if speed > MAX_SPEED:
    scale = MAX_SPEED / speed
    velocity_x *= scale
    velocity_y *= scale

Key Properties

  • Preserves direction while limiting magnitude
  • No velocity component exceeds MAX_SPEED (300 px/s)
  • Prevents unrealistic movement speeds
  • Ensures fair hunt/evade dynamics

Jitter Effect

Adds organic randomness by rotating velocity vector.

if random() < 0.05:  // 5% chance per frame
    speed = sqrt(velocity_x² + velocity_y²)
    current_angle = atan2(velocity_y, velocity_x)
    rotation = random(-30°, 30°)
    new_angle = current_angle + radians(rotation)
    
    velocity_x = speed * cos(new_angle)
    velocity_y = speed * sin(new_angle)

Characteristics

  • Preserves speed, only changes direction
  • 5% chance triggers rotation
  • ±30° rotation range
  • Creates unpredictable evasion patterns
  • Makes hunting challenging

Boundary Collision

if x < 0 or (x + size) > width:
    velocity_x *= -1

if y < 0 or (y + size) > height:
    velocity_y *= -1

Behavior

  • Simple velocity reversal on wall hit
  • Creates bouncing effect
  • Keeps agents within canvas
  • Adds complexity to movement patterns

Force Accumulation

Multiple threats or prey accumulate their forces:

Example: Prey with 3 nearby predators

Predator 1 (distance 50px):
  urgency = (100-50)/100 = 0.5
  force = 0.5 * 2.0 = 1.0

Predator 2 (distance 30px):
  urgency = (100-30)/100 = 0.7
  force = 0.7 * 2.0 = 1.4

Predator 3 (distance 80px):
  urgency = (100-80)/100 = 0.2
  force = 0.2 * 2.0 = 0.4

Total accumulated force = 1.0 + 1.4 + 0.4 = 2.8
Result: Much stronger evasion when surrounded
